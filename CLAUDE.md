# CLAUDE.md — netinetidesign

Personal portfolio and tech blog for Marcel Reig, a full-stack developer with a frontend focus.

**Live site**: https://www.netinetidesign.com

## Tech Stack

- **Static site generator**: Jekyll 4.3+ (Ruby)
- **Build pipeline**: Gulp 4 + npm
- **CSS**: Bootstrap 5.3, custom SCSS compiled via gulp-sass
- **JS**: jQuery 3.6, Bootstrap Bundle, Masonry Layout, Magnific Popup, jQuery Validation — all bundled into `assets/scripts/app.js`
- **Icons**: FontAwesome 6 (free)
- **Template language**: Liquid (Jekyll)

## Development Commands

```bash
# Install dependencies (first time)
bundle install
npm install

# Start dev server (port 4000, live reload via BrowserSync)
npm run gulp

# Production build (minified CSS/JS, JEKYLL_ENV=production)
NODE_ENV=production gulp

# Individual tasks
gulp build_styles     # Compile SCSS only
gulp build_scripts    # Bundle JS only
gulp build_jekyll     # Build Jekyll only
gulp clean_jekyll     # Delete _site/
gulp clean_assets     # Delete assets/
```

Dev build uses `_config.yml` + `_config_localhost.yml`. Production uses `_config.yml` only.

## Directory Structure

```
_assets/          # Source files (edit these, never assets/ directly)
  styles/         # SCSS source (main.scss imports everything)
  scripts/        # Custom JS (main.js and others)
  images/         # Source images
  fonts/          # Custom fonts
  cv/             # CV PDF source

assets/           # Compiled output (generated by Gulp, do not edit)
  styles/main.css
  scripts/app.js
  images/
  fonts/
  webfonts/       # FontAwesome
  cv/

_layouts/         # Jekyll page templates
  default.html
  post.html
  blog.html
  blog_by_category.html
  blog_by_tag.html

_includes/        # Reusable template partials
  head.html
  nav.html
  footer.html
  google-analytics.html
  disqus_comments.html

_data/            # YAML data files (drive dynamic content)
  site.yml        # General site metadata
  navigation.yml  # Nav menu structure
  projects.yml    # Portfolio projects
  skills.yml      # Technical skills list
  social.yml      # Social media links
  blog.yml        # Blog configuration

_posts/           # Blog articles (markdown, YAML front matter)
_my_categories/   # Category collection pages
_my_tags/         # Tag collection pages
_site/            # Jekyll build output (gitignored)
contact/          # Contact page
blog/             # Blog listing
portfolio/        # Portfolio pages
```

## Content Patterns

### Blog posts (`_posts/`)
Filename format: `YYYY-MM-DD-post-title.md`

```yaml
---
layout: post
title: "Post Title"
description: "Short description for SEO"
categories: javascript
tags: [es6, arrays]
image: assets/images/blog/image.webp
lang: es
---
```

### Jekyll config (`_config.yml`)
- Permalink: `/post/:title/`
- Paginate: 6 posts per page, path `/blog/page:num/`
- Categories collection: `_my_categories/`, permalink `/blog/category/:name/`
- Tags collection: `_my_tags/`, permalink `/blog/tag/:name/`

### Portfolio projects (`_data/projects.yml`)
Projects have `title`, `description`, `url`, `github`, `image`, `technologies[]`, `category`, `featured`, `status` fields.

## Build Pipeline Notes

- SCSS source: `_assets/styles/main.scss` → compiled to `assets/styles/main.css`
- JS bundle order: jQuery → jQuery Validation → Bootstrap Bundle → Masonry → Magnific Popup → `main.js`
- Images are copied with `gulp-newer` (only copies changed files)
- Production mode: CSS compressed, JS uglified (`NODE_ENV=production`)
- FontAwesome webfonts copied from `node_modules` to `assets/webfonts/`
